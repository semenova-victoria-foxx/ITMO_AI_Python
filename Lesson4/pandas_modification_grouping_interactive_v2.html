Создание, удаление и изменение столбцов

  • Новый столбец можно создать присваиванием: df['new_col'] = ....
  • Векторные операции обычно быстрее цикла for с iterrows().
  • Для изменения части строк лучше использовать loc.
  • replace удобно для замены значений по словарю.
users['Age_days'] = users['Age'] * 365
users = users.drop(columns='new_feature')
users.loc[users['Geography'] == 'France', 'target'] = 1
users['Gender'] = users['Gender'].replace({'Female':'F', 'Male':'M'})
Важно: запись вида df[df['cond']]['col'] = ... опасна. Для изменения исходной таблицы используйте loc.

Модификация строк: apply, lambda и строковые методы

  • Если в столбце лежат строки, их можно менять поэлементно через apply.
  • lambda удобно использовать для коротких правил преобразования.
  • Частые операции: replace, split, strip, lower, срезы строк.
  • Так можно убрать часть слова, взять фрагмент до символа-разделителя или получить логин из email.
df['name_short'] = df['name'].apply(lambda x: x.replace('ООО ', ''))
df['code_left'] = df['code'].apply(lambda x: x.split('-')[0])
df['login'] = df['email'].apply(lambda x: x.split('@')[0])
На вход лямбде приходит одна строка, на выходе — новая строка. Так создают или изменяют текстовые признаки.

Объединение таблиц: merge и join

  • merge — основной способ соединить таблицы по общему ключу.
  • left сохраняет все строки левой таблицы.
  • right сохраняет все строки правой таблицы.
  • inner оставляет только совпавшие ключи.
  • outer сохраняет все ключи из обеих таблиц.
merged = users.merge(bank, on='CustomerId', how='outer')
users_id = users.set_index('CustomerId')
bank_id = bank.set_index('CustomerId')
result = bank_id.join(users_id).reset_index()

Агрегация через agg

  • agg сводит много значений к нескольким итогам: минимум, максимум, среднее и т.д.
  • Можно агрегировать один столбец или сразу несколько.
  • Можно задавать собственные имена итоговым метрикам.
users['Age'].agg(['min', 'max'])
users.agg({
    'Age': ['min', 'max'],
    'EstimatedSalary': 'mean'
})
users.agg(
    min_age=('Age', 'min'),
    max_age=('Age', 'max'),
    mean_salary=('EstimatedSalary', 'mean')
)

Группировки: groupby

  • groupby разбивает строки на группы по значению столбца.
  • После этого обычно применяют agg, sum, mean, count.
  • Это способ ответить на вопросы вида “что происходит по странам / клиентам / категориям?”.
grouped = toy_df.groupby('client_id')
grouped.agg({'price': ['sum', 'min', 'max']})
users.groupby('Geography').agg({'Age': ['mean'], 'EstimatedSalary': ['min']})

Сводные таблицы: pivot_table

  • pivot_table группирует данные и раскладывает одну категорию в строки, другую — в столбцы.
  • Подходит для компактных сводок по категориям.
  • Параметр margins=True добавляет итоги.
pivot = pd.pivot_table(df,
    values='Sales',
    index='Product',
    columns='Region',
    aggfunc='sum')

users.pivot_table(index='Geography',
                  columns='Gender',
                  values='EstimatedSalary',
                  aggfunc='mean',
                  margins=True)

Перекрёстные таблицы: crosstab

  • crosstab показывает частоты сочетаний категорий.
  • Можно считать не только количество, но и агрегаты по values + aggfunc.
  • normalize переводит частоты в доли.
pd.crosstab(index=users['Geography'], columns=users['Gender'])

pd.crosstab(index=users['Geography'],
            columns=users['Gender'],
            normalize='index')
Если нужен именно подсчёт сочетаний категорий, crosstab часто читается проще, чем ручной groupby.
